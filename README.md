# Easy Barter - 现代易货交易平台

[![CI](https://github.com/sxc19960907/neobarter/actions/workflows/ci.yml/badge.svg)](https://github.com/sxc19960907/neobarter/actions/workflows/ci.yml)

## 一、项目概述

### 1.1 产品名称
**新易巴特（NeoBarter）** - 现代易货交易平台

### 1.2 产品定位
| 维度 | 说明 |
|------|------|
| **目标用户** | C2C（个人用户）+ B2B（企业用户）双模式 |
| **平台形态** | 先实现**网页应用**（响应式设计，兼顾移动端），后续扩展APP |
| **核心价值** | 让闲置物品流动起来，实现"以物易物"的现代升级 |

### 1.3 项目背景
原版易巴特（eBarter）于2002年在青岛成立，是中国首家现代易货交易专业机构。平台通过互联网让古老的"以物换物"焕发新生，帮助企业解决资金短缺、产能过剩、库存积压等问题。

新易巴特在此基础上，面向个人和企业用户，打造现代化的C2C+B2B易货交易平台。

## 二、功能设计

### 2.1 原版核心功能（继承）

| 功能 | 说明 |
|------|------|
| 易货信息发布 | 用户发布可交换物品 |
| 易货查询/搜索 | 按类别、地区、价值筛选 |
| 交易结算 | 虚拟货币（巴特币）结算系统 |
| 交易安全监控 | 平台担保交易 |
| 多边易货 | 支持A→B→C多边交换 |

### 2.2 创新功能（新增）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **AI智能匹配** | 基于用户需求和物品价值，自动推荐交换对象 | P0 |
| **双模式切换** | 个人模式（C2C）/ 企业模式（B2B）一键切换 | P0 |
| **物品估值系统** | AI评估物品市场价值，辅助交换定价 | P1 |
| **信誉体系** | 交易评价、芝麻信用接入、实名认证 | P0 |
| **在线沟通** | 内置IM即时通讯，支持图片、语音 | P0 |
| **物流对接** | 对接顺丰/京东物流，自动生成运单 | P1 |
| **视频验货** | 发布物品时可上传短视频展示状态 | P1 |
| **积分商城** | 交易积分可兑换平台权益 | P2 |
| **社区广场** | 换物故事、好物推荐、换客圈子 | P2 |
| **企业专区** | 批量发布、合同模板、发票对接 | P1 |

## 三、技术架构

### 3.1 技术栈选型

| 层级 | 技术选择 | 理由 |
|------|----------|------|
| **前端** | React 18 + TypeScript + Ant Design | 生态成熟、组件丰富、类型安全 |
| **后端** | Go (Gin) 或 Node.js (NestJS) | Go性能极高；NestJS可维护性强 |
| **数据库** | PostgreSQL + Redis | 关系型+缓存组合，稳定可靠 |
| **搜索引擎** | Elasticsearch | 物品全文搜索、智能匹配 |
| **AI服务** | Python FastAPI (估值/匹配算法) | 独立微服务，便于迭代 |
| **即时通讯** | WebSocket + RabbitMQ | 实时消息推送 |
| **部署** | Docker + K8s + 云服务器 | 容器化部署，易扩展 |

## 四、MVP范围（第一阶段）

### 核心闭环

```
用户注册/登录 → 发布物品 → 浏览/搜索 → 发起交换 → 在线沟通 → 确认交易 → 评价
```

### MVP功能清单

- [ ] 用户注册/登录（手机号+验证码）
- [ ] 个人/企业身份选择
- [ ] 发布物品（图片+描述+期望交换物）
- [ ] 物品浏览/搜索/筛选
- [ ] 发起交换请求
- [ ] 在线IM沟通
- [ ] 交易确认/完成
- [ ] 基础信誉评价
