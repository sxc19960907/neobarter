# 实现前端 React + TypeScript + Ant Design 应用

## Goal

实现 NeoBarter 平台的 Web 前端应用，覆盖 SRS 文档中所有 P0/P1 功能需求，对接已完成的后端 API。

## Requirements

### 技术栈
- React 18 + TypeScript 5
- Ant Design 5 组件库
- React Router 6 路由
- Zustand 状态管理
- Axios HTTP 客户端
- Socket.io-client WebSocket
- Vite 构建工具

### 页面模块
1. **认证模块** — 手机号+验证码登录/注册，用户类型选择
2. **首页** — 推荐物品列表，分类导航，搜索入口
3. **物品模块** — 物品浏览（列表/详情）、发布、管理（编辑/上下架/删除）
4. **交易模块** — 发起交换、交易列表、接受/拒绝/完成交易
5. **消息模块** — 会话列表、聊天界面（文字/图片）、实时推送
6. **钱包模块** — 巴特币余额、交易流水
7. **个人中心** — 用户信息编辑、地址管理、我的物品、我的评价
8. **通知模块** — 通知列表、未读标记

### 设计要求
- 响应式布局（移动端优先，320px - 2560px）
- 中文界面
- 统一的加载/空状态/错误状态处理
- 图片上传预览
- 路由守卫（未登录跳转登录页）

## Acceptance Criteria

- [ ] 所有页面可正常渲染，无 TypeScript 编译错误
- [ ] 登录/注册流程完整可用
- [ ] 物品发布、浏览、搜索功能正常
- [ ] 交易发起→确认→完成流程无阻断
- [ ] 消息收发实时可靠
- [ ] 钱包余额和流水正确显示
- [ ] 移动端和桌面端布局均正常
- [ ] 与后端 API 对接无误（基于 docs/API.md）

## Constraints

- 前端项目位于 `web/` 目录（与 `server/` 同级，在项目根目录下）
- API Base URL 通过环境变量配置
- 开发环境使用 Vite proxy 代理到后端 8080 端口
